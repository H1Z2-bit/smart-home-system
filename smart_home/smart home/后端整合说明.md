# 后端整合说明

## 1. 整合来源

- 本地后端目录：`backend/`
- 组员仓库：`https://github.com/H1Z2-bit/smart-home-system.git`
- 本地拉取目录：`team_remote_smart_home_system/`

## 2. 已整合模块

本次已将组员后端中的以下模块整合进当前 FastAPI 后端：

- 用户认证：注册、登录、退出、用户资料、修改密码
- JWT 鉴权：Bearer Token 解析、当前用户获取
- 家庭空间：创建、查询、修改、删除家庭空间
- 成员管理：邀请、申请、同意邀请、审批、权限修改、移除成员
- 权限控制：OWNER、MEMBER、GUEST、MAINTAINER 等角色权限
- 系统配置：烟雾阈值、燃气阈值、高温阈值、自动报警、模拟开关
- 系统操作日志：登录、家庭、成员、系统配置相关日志
- SQLAlchemy 基础目录：`app/models/`、`app/db/`
- RSA 密钥生成脚本与后端 smoke test 脚本

你原本负责的以下模块也已保留：

- 房间管理
- 设备管理
- 设备控制
- 设备模拟
- 情景模式
- 定时任务
- 联动规则
- 报警处理
- 设备自检
- 设备/房间/情景等操作日志

## 3. 关键适配

两边都存在日志模块，直接合并会产生命名和接口冲突，因此做了隔离：

- 设备侧操作日志仍使用：`GET /api/homes/{home_id}/logs`
- 账号/成员/系统配置日志改为：`GET /api/homes/{home_id}/system/logs`

组员代码中的账号日志服务没有覆盖本地 `operation_log_service.py`，而是改为：

- `backend/app/services/account_operation_log_service.py`
- `backend/app/repositories/mock/account_operation_log_mock.py`
- `backend/app/repositories/mock/account_store.py`

这样后续可以分别演进：

- 设备业务日志对接设备管理、报警、联动模块
- 系统日志对接认证、成员、系统配置模块
