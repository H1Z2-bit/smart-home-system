# 智能家居综合管理系统完整方案与小组分工

## 一、项目名称

智能家居综合管理系统

## 二、项目定位

本项目面向家庭住宅场景，开发一套可完整使用的智能家居管理软件。系统通过软件方式模拟智能开关、智能插座、传感器节点等设备，实现用户注册登录、家庭空间管理、成员权限管理、设备管理、设备控制、情景模式、定时任务、联动控制、安防报警、设备自检、系统设置和日志记录等功能。

本系统不是单纯原型页面，而是包含前端界面、C++ 后端服务、MySQL 数据库、RESTful API、JWT 身份认证、角色权限控制和设备模拟机制的完整软件系统。

## 三、开发模型

本项目采用：

```text
增量迭代模型为主，原型模型为辅
```

| 模型 | 使用位置 | 作用 |
|---|---|---|
| 增量迭代模型 | 整体开发过程 | 将系统拆成多个可运行版本，逐步完成核心功能 |
| 原型模型 | 前端页面和流程确认 | 先设计登录、首页、设备控制、报警等页面原型，再完善功能 |

采用该模型的原因是：智能家居系统功能较多，涉及用户、权限、设备、报警、数据库、前后端联调等内容。采用增量迭代开发，可以先完成基础功能，再逐步扩展设备控制、情景联动、报警闭环等复杂功能，每一轮都形成可测试、可演示的系统版本，从而降低开发风险。

## 五、系统架构

系统采用：

```text
B/S 架构 + 前后端分离 + MVC 分层设计
```

技术结构为：

```text
浏览器
-> Vue3 + Element Plus 前端
-> RESTful API
-> C++ 后端服务
-> MySQL 数据库
```

系统内部采用分层架构：

| 层次 | 技术或内容 | 作用 |
|---|---|---|
| 表现层 | Vue3 + Element Plus | 展示页面，接收用户操作 |
| 控制层 | C++ Controller | 接收前端 RESTful API 请求 |
| 业务层 | Service | 处理登录、权限、设备、报警、情景模式等业务 |
| 数据访问层 | DAO / Model | 操作 MySQL 数据库 |
| 数据层 | MySQL | 保存系统全部业务数据 |

## 十四、数据库关系设计

核心关系如下：

```text
一个用户可以加入多个家庭空间；
一个家庭空间可以包含多个成员；
一个家庭空间可以包含多个房间；
一个房间可以包含多个设备；
一个设备可以产生多条传感器数据；
一个情景模式可以包含多个设备动作；
一个联动规则可以根据传感器状态触发设备动作；
一条报警记录可以对应多条报警处理日志；
一个设备可以对应多条自检记录；
一个用户可以产生多条操作日志。
```

## 十五、安全方案

系统安全采用：

```text
JWT + RSA + RBAC + 密码加盐哈希
```

| 安全机制 | 说明 | 负责人 |
|---|---|---|
| JWT | 用户登录成功后生成 token | 韩喆 |
| RSA | 对 JWT 签名，防止伪造 | 韩喆 |
| 密码加盐哈希 | 密码不明文存储 | 韩喆、王涵 |
| RBAC | 按角色控制权限 | 韩喆 |
| 操作日志 | 记录敏感操作 | 韩喆、周敬琦、王涵 |
| 前端 token 携带 | 请求头携带 Authorization | 邓延霄 |

登录流程如下：

```text
用户输入账号密码
-> 前端发送登录请求
-> 后端查询数据库
-> 后端验证密码哈希
-> 生成 JWT
-> 使用 RSA 私钥签名
-> 返回 token
-> 前端保存 token
-> 后续请求携带 token
-> 后端验签并校验权限
```

## 十六、设备模拟方案

设备模拟由周敬琦主要负责，王涵负责数据库支持。

模拟流程如下：

```text
device 表保存设备当前状态

报警模块判断是否超过阈值
超过阈值则生成报警记录
如果存在联动规则则自动执行设备动作

```

|---|---|

| 温度传感器 | 当前温度 |
| 烟雾传感器 | 烟雾浓度、正常/异常 |
| 燃气传感器 | 燃气浓度、正常/异常 |
| 门磁传感器 | 打开、关闭 |

## 十七、报警闭环设计

报警流程如下：

```text
传感器数据异常
-> 系统生成报警记录
-> 前端报警中心显示报警
-> 用户查看报警详情
-> 户主确认报警
-> 户主填写处理结果
-> 系统进行复检
-> 复检正常后消警
-> 保存报警处理日志
```

报警状态建议：

```text
未确认
已确认
处理中
待复检
已消警
误报
```

## 二十、报告可直接使用的总结

本项目采用增量迭代模型为主、原型模型为辅的开发方式。系统采用 B/S 架构和前后端分离设计，前端使用 Vue3 和 Element Plus，后端采用 C++ Web 框架提供 RESTful API，数据库使用 MySQL 保存系统数据。系统内部采用分层架构和 MVC 设计思想，以降低模块耦合，提高可维护性和可扩展性。

在安全方面，系统采用 JWT 进行用户身份认证，使用 RSA 对 token 进行签名，并结合 RBAC 角色权限控制区分户主、常住成员、临时成员和维护人员的操作范围。用户密码采用加盐哈希方式保存，避免明文存储。

由于课程设计不要求接入真实硬件，系统通过后端定时任务和模拟设备状态表实现智能设备和传感器节点的软件模拟，从而支持设备控制、情景模式、定时控制、联动控制、安防报警和设备自检等完整功能。
